48915: Character movement restriction data
The values in this table serve to restrict the movement of characters in two ways:
  • when a character tries to move left or right within a room, his / her new x-coordinate is checked against the two values in the entry below for that character's current room. If the new x-coordinate is equal to either of the current room's values here, then the move is aborted and the routine that controls the character's movement (62622) is returned from.
  • when a character tries to move into a new room, the first byte in the entry for the room the character is trying to enter is checked. If this value is 98, then the move is prevented. Instead of adding double the index of the destination room to the start address of this table (48915), double the index of the character's current room is added to either 48913 (when moving left) or 48917 (when moving right). This is equivalent, however, because the entries in this table are ordered such that the rooms run left-to-right.
